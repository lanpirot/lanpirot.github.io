Research

Automated merge conflict resolution

When developers work on the same code in parallel, their changes eventually have to come back together, and today's three-way textual merge still hands too many conflicts back to humans. I study where current tools fail, build techniques that resolve conflicts semantically rather than line-by-line, and measure what developers actually encounter in the wild. The goal is a world where merging is something you notice only when it's genuinely ambiguous.

From merging to software variability

A branch is just a software variant that hasn't been integrated yet. That observation connects two fields that usually talk past each other: version control and software product line engineering. I work on carrying results across the bridge — reusing variability-aware analyses to understand long-lived branches, and feeding insights from real merge histories back into how we reason about families of software variants.

Model-based software engineering with MATLAB/Simulink

Empirical research on MATLAB/Simulink is starved for data: industry rarely shares the models that academic tools need to be evaluated on. My work tackles this head-on by collecting and indexing publicly available models so researchers can find them, synthesizing realistic models where no public ones exist, and developing techniques to strip sensitive content from industrial models so companies can share them safely. The underlying goal is to make industry–academia collaboration in model-based software engineering actually possible.
